# Переезд с Python на разные языки — план и статус

Оригинал (корень репозитория) — рабочий Python-бот, ничего в нём не сломано, можно
разворачивать как раньше. Здесь — постепенный перенос отдельных частей на другие языки,
каждая часть независимая, переносится по частям.

## Статус

| Часть                          | Было (оригинал)          | Стало                | Статус |
|---------------------------------|---------------------------|----------------------|--------|
| Ядро бота (команды)              | py-cord                  | TypeScript, discord.js | `/rewrite/bot` — рабочий скелет: `/ping`, `/profile` (упрощённая схема игрока), подключение к Mongo, юнит-тесты на форматирование embed'ов проходят |
| Сервер генерации картинок         | FastAPI + Pillow          | Rust, axum + image + ab_glyph | `/rewrite/image-service` — рабочий HTTP-сервис, эндпоинт `/generate/profile` реально генерирует PNG (проверено сборкой и живым запросом) |
| База данных                      | MongoDB (motor/pymongo)   | MongoDB остаётся, у каждого сервиса свой нативный драйвер | не отдельная часть — это аспект каждого сервиса, не самостоятельный сервис |
| Веб-панель (nicegui)              | Python                   | не начато             | следующее на очереди, скорее всего TypeScript/React — отдельный веб-сервис проще нового фронта, чем py-обёртки nicegui |
| Разбор реплеев                    | Python-обёртка над бинарником | без изменений    | `lib/replay_parser/parser.py` в оригинале — это просто `subprocess` вызов уже скомпилированного `bin/parser`. Сам бинник — не Python, исходников нет. Переписывать нечего, кроме тонкой обёртки; она бы вызывала тот же бинарник что на TS, что на Go — без разницы. Оставил как есть. |
| Бэкап/дамп Mongo                  | Python (`workers/db_backup_worker.py`, `lib/dump_handler/`) | не начато | мелкая утилита, низкий приоритет |

## Почему так, а не иначе

- **Discord-бот → TypeScript**: discord.js — самая зрелая экосистема для slash-команд,
  embed'ов, кнопок/модалок/селектов (в оригинале это `lib/views/*` — довольно много
  интерактивных элементов). Меньше кода на то же самое, чем на Go/Rust-либах для Discord.
- **Картинки → Rust**: чисто CPU-bound задача (рисование поверх фона), Rust здесь даёт
  предсказуемую скорость и лёгкий деплой одним бинарником. Проверял и Go: набор пакетов
  под шрифты/растеризацию текста для Go недоступен в текущем окружении (сеть до
  proxy.golang.org отсутствует), тогда как реестр crates.io для Rust — доступен и
  реально собрался. Поэтому Rust, не Go — не вкусовщина, а то, что реально можно было
  собрать и проверить здесь и сейчас.
- **БД без общего слоя**: в моно-репо с разными языками нет смысла тащить общий
  ORM/слой — каждый сервис говорит с MongoDB на своём нативном драйвере
  (`mongodb` для Node, позже что-то вроде `mongodb`/`bson` crate для Rust, если БД
  понадобится и в image-service).

## Как запускать новые куски

```
cd rewrite/bot && cp .env.example .env  # заполнить DISCORD_TOKEN, MONGODB_URI
npm install
npm run dev

cd rewrite/image-service
cargo run   # слушает :8080, POST /generate/profile {nickname, region, battles, winrate, avg_damage}
```
